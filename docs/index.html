Lean Claude Code.
Less context. Same power.

Cut startup tokens to ~4.5–6.5k. Keep six essential tools. Pick Ultra or Regular lean — or install both and choose per session.

One-command install
curl -fsSL https://raw.githubusercontent.com/0p9b/claude-code-lean/main/install.sh | bash

Interactive menu with confirm step · OAuth still works · No API key required

See install options View on GitHub

Three ways to install

Same lean settings and six tools every time. Only the system prompt differs.

1

Ultra Lean

~4.5–5k tokens

Settings + claude-lean launcher with a minimal custom system prompt.

Run: claude-lean

2

Regular Lean

~6.5k tokens

Settings only. Uses Claude Code’s default lean system prompt.

Run: claude

Recommended 3

Both

Best of both

Install everything. Pick claude-lean or claude when you launch.

Run either command

Skip the menu (non-interactive)

curl -fsSL https://raw.githubusercontent.com/0p9b/claude-code-lean/main/install.sh | CLAUDE_LEAN_MODE=both bash

What you get

Opinionated defaults for a smaller, faster Claude Code session.

Enabled

Bash · Read · Write · Edit · WebSearch · WebFetch

Disabled

MCP, skills, workflows, agents, plan mode, cron, tasks, LSP, auto memory, and more

Defaults

Effort medium · OAuth login · No --bare

Safe install

Backs up existing settings.json before overwriting

After install

1

Restart any open Claude Code sessions

2

Run claude-lean or claude

3

Type /context to verify token usage

Requires Claude Code CLI and curl. If claude-lean isn’t found: export PATH="$HOME/.local/bin:$PATH"
